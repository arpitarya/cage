# Handoff: `[sources]` in policy — configurable import paths per agent/tool

**One-liner:** Implement plan Phase 4: a `[sources]` policy table that adds (or replaces) the log locations `cage import` probes — one or more paths per agent, plus custom tools reusing a declared parser format — with env > policy > built-in precedence, provenance in `doctor --paths`, and a portability guard for machine paths in committed policies.
**Owner / executor:** Claude Code (cage repo)
**Status:** Ready to build
**Stress-tested:** Debate recorded in the plan (Phase 4). Survivors: (a) paths in a *committed* project policy are machine-specific by nature → `~`-expansion supported, doctor's portability check warns, docs steer `[sources]` to the global `~/.cage/policy.toml`; (b) "custom tool = custom parser" scope trap → a custom source must declare `format = "claude|codex|copilot|kiro"` and reuse that parser — new formats are out of scope by construction; (c) merge-vs-replace ambiguity → additive by default, `replace = true` per table for the rare clean override; (d) `policy sync` must treat `[sources]` as user-owned (bundle ships none — assert). Residual risk: a bad user path silently yields zero rows — mitigated because that's exactly what `doctor --paths` provenance and the debug log already surface.

## 1. Context & background

Log locations live in `paths.agent_log_sources()` (per-OS candidates) with env overrides (`CLAUDE_CONFIG_DIR`, `CODEX_HOME`, `COPILOT_HOME`, `KIRO_HOME`) as the only user knob. Field ask: expose the probe paths in `policy.toml` for one or more tools — nonstandard installs, network homes, side-by-side log copies, and custom emitters that write an already-supported format.

## 2. Definition of done

- [ ] **Schema:** `[sources.<agent>] paths = ["~/...", ...]` for the four agents (extends candidates); optional `replace = true` (that table's built-ins ignored); custom tools: `[sources.<name>] paths = [...], format = "claude|codex|copilot|kiro"` (agent stamped as `<name>` on imported rows — additive, PII rules unchanged); `~` and env-var expansion in paths.
- [ ] **Precedence:** env override > policy `[sources]` > built-in registry — resolved in ONE place (`paths.agent_log_sources()` grows a policy parameter; no second resolution path anywhere).
- [ ] **`cage doctor --paths`:** every candidate row names its provenance — `built-in | policy | env` — and custom-tool sources appear as their own sections; zero-match policy paths show the same `why` treatment as built-ins.
- [ ] **Portability guard:** doctor warns when a committed project policy carries non-`~`, machine-absolute `[sources]` paths ("teammates' clones will probe a path that doesn't exist — move to ~/.cage/policy.toml or use ~"); global-policy sources never warn.
- [ ] **Import behavior:** policy sources sweep with the same cursors/dedupe/fail-open as built-ins (a missing path is a debug-logged skip, never an error); export's import-first sweep includes them; `CAGE_CAPTURE=0` disables them with everything else.
- [ ] **`policy sync`:** `[sources]` is user-owned — never added/updated/orphan-warned (bundle ships none; asserted by test).
- [ ] `cage query sources` concept entry (schema, precedence, portability rule, live current sources); `docs/pricing.md`-style section in the right doc (plan or a short `docs/sources.md` — executor picks, indexes it); README one-liner in the capture section; CHANGELOG in-tree; CLAUDE.md proposed-not-applied; skillgen if CLI text changed.
- [ ] Tests: precedence matrix (env beats policy beats built-in; replace semantics), expansion, custom-tool import end-to-end (fixture log in a custom path → rows with agent=`<name>`), cursor incrementality on policy paths, portability warn/no-warn, sync ownership, doctor provenance goldens. Dummyrepo: one scenario step importing from a policy-declared path. `just test` green. Nothing committed.

## 3. Scope

**In scope:** the table, resolution, doctor provenance, guard, docs/query.
**Out of scope (explicit):** new log formats or parser changes; per-source policy overrides beyond paths+format (no per-source prices/rates); remote/network fetch (paths are local filesystem, same as ever); auto-discovery beyond the declared paths.

## 4. Current state

## 5. Technical approach (decided)

`agent_log_sources(pol=None)` returns candidate lists already tagged with provenance; import and pathprobe both consume the tagged form (one resolution, two renderers — the freshness pattern). Custom tools ride the existing per-format parse functions keyed by the declared `format`; rows stamp `agent=<name>` so reports split them naturally. Cursors key on resolved absolute path (existing mechanism) — policy paths get incrementality for free.

## 6. Non-negotiables / constraints

One resolution point — no view or importer re-derives sources · fail-open sweep (bad path = debug-logged skip) · additive schema, empty `[sources]` = today's behavior byte-identical · PII rules unchanged for custom tools (counts only; the format parsers already enforce it) · determinism of derived views untouched (sources affect *capture*, never derivation) · $0/stdlib · no commits/tags/publishes.

## 7. Dependencies & prerequisites

`policy-sync` must be in the tree (the ownership rule lands there-or-here consistently); benefits from `cli-tiering` landing first only for hint-string stability — order with it per the plan queue. STOP and ask if the queue state is unclear.

## 8. Edge cases & risks

Duplicate path declared for two agents → both probe it; dedupe-by-id makes double-import safe, but doctor flags the overlap (likely user error) · glob-shaped entries (`*`) → not supported v1; validated out with a typed error naming the rule · policy path == a built-in path → deduped at resolution, provenance shows `built-in` · custom tool named like a real agent (`[sources.claude]` is the AGENT table; `[sources.claude2] format="claude"` is a custom tool — the four agent names are reserved keys) · empty `paths = []` with `replace = true` → that agent's capture disabled by policy; doctor says so plainly (legitimate use: silencing a never-installed agent's probe noise).
