# Claude Code prompt: capture health — make silent zero-capture loud

You are implementing `docs/capture-health.handoff.md`: when an agent is installed but its log
source matched nothing, `cage report` says so — instead of printing confident totals from the
agents that still work while a whole agent is silently capturing zero.

Read the handoff first, including the **Stress-tested** header (it records what the debate gate
cut and why). Its Definition of Done (§2), Scope (§3) and Non-negotiables (§6) are binding.

## The one thing that makes or breaks this

**A false-positive nag poisons the whole footer.** UNPRICED ⚠, method tags and caveats live
there and are load-bearing. One warning that fires wrongly trains the user to skim past all of
them. The feature is only safe because of the **triple gate** — warn for agent A only when
**all three** hold:

1. **Installed** — A's home marker exists (`~/.claude`, `~/.codex`, `~/.copilot` or the VS Code
   user dir, `~/.kiro` or the Kiro data dir). Absent ⇒ **silent forever.**
2. **Nothing found** — at the last import, every source for A matched **0 files**.
3. **Never captured** — A has contributed **zero rows** to the ledger, ever.

Clause 3 makes the warning unreachable for any agent that has ever worked — it self-silences
after one captured row. **Do not weaken any of the three.** If you find yourself wanting to,
stop and tell me.

## Do NOT build these (cut by the gate — handoff §3)

- **A live filesystem probe on any read path.** New I/O on a hot path, and it would make
  `render_report` impure. It is currently a pure function of its args (`clicmds.py:55` reads
  the state and passes it in) — **that contract holds.**
- **A new state file.** `cursors.json` already carries `_last_import` (`importcmd.py:372`);
  `_`-prefixed metadata in the cursor map is the established precedent. Use `_health`.
- **A new opt-out config key.** `[sources.<agent>] replace=true, paths=[]` already means
  "disabled by policy" and is already documented. Reuse it.
- Any change to the tables, to CSV, or to a derived number. **Footer only.**
- Guessing, inferring, or scanning for a path. Cage never guesses a location.
- Forking `pathprobe._why()`'s strings. Reuse the vocabulary.

## Traps that will bite you (handoff §8)

- **Copilot has two sources** (CLI + VS Code) with two different homes. Gate 2 needs *all* its
  sources at 0; gate 1 passes if *either* home exists. Get this wrong and every CLI-only
  Copilot user gets nagged.
- **Kiro is a file source** — `_scan` sets `raw = [src]` when `src.is_file()`, so `len(raw)` is
  1 even for an empty file. Gate 2 must mean "no data", not "no file". Kiro's log is coarse by
  design (`generatedTokens` often 0) — that is **not** broken.
- **`disabled by policy`** ⇒ the agent has no sources at all ⇒ silent, not "0 files matched".
  Check `SourcesResolution.disabled`.
- **`cleanup`'s orphan-cursor prune** must not eat `_health`. `_last_import` survives today —
  find out why and inherit it. Test it.
- **Fresh install, never imported** ⇒ no `_health` ⇒ no warning. Correct. For `cage doctor`'s
  fallback see the handoff §10 open question — **ask me before choosing.**

## Constraints (hard)

- **Stdlib only.** `dependencies = []`.
- **Fail-open on the write path.** A health-write failure never breaks an import.
- **`render_report` stays pure** — no filesystem access from `report.py`. State is read in
  `clicmds` and passed in.
- **Determinism:** same ledger + same policy ⇒ same **tables**. The footer may vary with
  machine state (`_last_import` already does); the tables may not.
- **CSV never gates** — no warning text in any `--csv` output.
- **Four agents, always.**
- **New swallow site ⇒ a `CAGE_DEBUG` log.** `tests/test_debug_coverage.py` enforces this.
- **Do not touch:** `pathprobe._why()`'s strings; the `NEVER` list; the tables.
