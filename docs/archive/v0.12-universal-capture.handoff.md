# Handoff: universal capture — explicit `import`/`export` over a global ledger

**One-liner:** Make cage capture work for any agent, any client, and the no-project user by
making explicit `cage import` (capture) and `cage export` (import-then-emit) the primary
path over a global ledger, with "project" as a derived view via a new `project` field.
**cage installs no background/OS job** — automation, if wanted, is the user's own cron line
calling `cage import`.
**Owner / executor:** Claude Code
**Status:** Ready to build
**Stress-tested (two passes):** Pass 1 — (1) global-default vs per-repo isolation → kept
project-local `.cage/` as opt-in override + `CAGE_LEDGER` relocation; (2) per-project
attribution is Claude-only today → dedicated `project` field, not a `scope` overload;
(3) background-scheduler footprint → cut entirely, no OS job; (4) `export` import-first
side effect → kept for freshness, made visible + `--no-import`. Pass 2 (on the final doc) —
(5) **`append_new` reloads the whole ledger + import re-scans every log each run (ledger
already 22k+ rows); with no daemon the hot paths are manual import and `cage watch` looping
→ O(all logs × whole ledger) per run. FIX: incremental import via a per-agent high-water
cursor (§5).** (6) **Global ledger = one unbounded JSONL → must inherit the §3.6.1 month
partitions + `--since` defaults.** (7) **No daemon means capture only on user action — a
Copilot-only user who never runs `import` sees an empty ledger (the original complaint,
reshaped). FIX: doctor/report show "last import: N ago" + nudge; docs frame cage as
pull-based, not a passive meter.** **Residual risk to watch:** per-project numbers exact for
Claude only (don't imply precision for Copilot/Kiro); capture is on-demand by design — the
staleness nudge is what keeps that from reading as a bug.

## 1. Context & background
cage is a package any user installs. They may use only Copilot, only Codex, only Kiro, or
any mix — in a standalone CLI **or a VS Code extension**. Field-proven this session: hooks
are client-specific and mostly don't fire (VS Code extensions of Codex/Kiro never execute
`.codex/hooks.json` / `.kiro/hooks/*.hook`; only Claude Code's extension honors its hooks),
yet `cage doctor` reports them "wired." Meanwhile the on-disk import works for all four,
always (`cage import --agent all` captured 22322 Claude + 212 Codex + 10 Copilot + 7 Kiro).
The fix is to lead with explicit import/export, not hooks — and to install nothing in the
background.

A second forced fact: the non-Claude usage logs carry **no project identity** — Copilot
(`~/.copilot/session-state/<id>/events.jsonl`) and Kiro (single global
`tokens_generated.jsonl`) have no cwd/project, and the Codex parser extracts none. Only
Claude's log is project-mapped (via `~/.claude/projects/<slug>/`). Therefore project-level
*capture* is impossible for Copilot/Kiro — project can only ever be a *derived attribution
view*, exact where the log supports it.

## 2. Definition of done
- [ ] **`cage import`** is the canonical explicit capture verb — sweeps every agent's
  on-disk log into the ledger (`--agent all` default), idempotent, prints a per-agent count,
  no other output. Works with no hooks and no project `.cage/`. **Uses a per-agent
  high-water cursor** so it ingests only new data (not the whole world) each run.
- [ ] The **global ledger is month-partitioned** (§3.6.1 dated shards), never one unbounded
  file; `cage doctor`/`cage report` show **"last import: N ago"** and nudge when stale.
- [ ] **`cage export`** runs import first (refresh), then emits the ledger in a portable
  format: `--format jsonl|csv|json`, `--since`, `--project`, `--agent`, `-o FILE` (else
  stdout), and `--no-import` to skip the refresh. Prints `↻ imported N new call(s)` before
  emitting so the side effect is visible.
- [ ] A user with **no `.cage/` project** captures into a **global ledger** (`~/.cage` or
  `CAGE_LEDGER`); resolution precedence: `--ledger`/`CAGE_LEDGER` → nearest project `.cage/`
  from cwd → global ledger.
- [ ] With **no hooks at all**, running `cage import`/`cage export` while using **any single
  agent** (e.g. Copilot only) yields that agent's spend in `cage report`/the export.
- [ ] A new additive optional `project` field is stamped on Claude imports (dir basename);
  `cage report --project <name>`/cwd-basename filters the global ledger to a project view.
  `scope` (§3.6.2) is untouched. No per-repo physical ledger is required for capture.
- [ ] **`cage watch`** — an optional **foreground** poll loop (`import_all`; `sleep
  interval`; Ctrl-C clean) for users who want a running loop they control. It is NOT an OS
  job and registers nothing.
- [ ] **cage installs no OS scheduler** (no launchd/systemd/schtasks, no `cage scheduler`
  command). `cage doctor` may *mention* that a user can add their own cron calling
  `cage import`, but cage never creates or manages it.
- [ ] `cage doctor` reports the **actual** capture method per agent and **warns hooks don't
  fire under VS Code extensions**; it no longer labels an unfireable hook "capture wired,"
  and points at `cage import`/`cage export` as the universal path.
- [ ] Hooks, MCP, and the existing project-local `.cage/` ledger still work unchanged.
- [ ] Documentation updated to match (see §9.5).

## 3. Scope

**Out of scope (explicit):** **any OS/background scheduler (launchd/systemd/schtasks) or a

removing or redesigning hooks (only demote them); retiring the project-local `.cage/` ledger
(keep it as an opt-in override).

## 5. Technical approach (decided)
- **Explicit `import`/`export` are the primary capture UX.** `import` = capture only.
  `export` = `import` (unless `--no-import`) then serialize the ledger. Three formats:
  `jsonl` (raw rows, lossless re-ingest), `csv` (flat, for spreadsheets/BI), `json`
  (structured summary: totals by agent/model/project). Honors `--since`/`--project`/`--agent`
  and `-o`/stdout. Reuse `report.py`'s aggregation for the `json` summary; don't duplicate.
- **Global ledger is the default capture sink.** One active sink per run (no double-write):
  `--ledger`/`CAGE_LEDGER` → nearest project `.cage/` from cwd → global (`~/.cage`). All
  read/emit surfaces honor the same resolution.
- **Project is a derived view via a NEW `project` field.** Do **not** overload `scope`
  (§3.6.2 = top-level changed dir in a monorepo — a different axis). Add a dedicated
  **additive optional `project`** field (empty = legacy contract; **basename only, never a
  full path** — same PII guard as `scope`/tasks). Populate it from the **confirmed `cwd`
  field**, not the lossy slug: **Claude now** = `basename(rec['cwd'])` read off each assistant
  record (confirmed present); **Codex/Copilot** = `basename(cwd)` **only if** their log
  carries it (verify `session_meta`/`turn_context` + the Copilot session events — leave empty
  until confirmed); **Kiro** = always empty. `cage report --project <name>`/cwd-basename
  filters; `--scope`
  stays §3.6.2. Accepted edge: two dirs sharing a basename merge under one `--project` value
  (rare, single-machine; a later enhancement can disambiguate without changing the field).
- **Incremental import (required for scale) — file-stat skip + once-per-run dedupe.**
  `cage import` must not re-process the whole world each run. (a) Keep a per-file `(size,
  mtime)` cursor in `.cage/state/` and **skip unchanged files entirely**; (b) build the
  ledger id-dedupe set **once per run**, not once per `append_new` call. `append_new`'s
  id-dedupe stays the correctness backstop, not the throughput strategy. This makes manual
  import, `export`'s import-first, and `cage watch` cheap as the ledger grows (already 22k+
  rows). (Per-file byte-offset parsing is a deliberate non-goal here — it forces partial-file
  reads through every parser; revisit only if one ever-growing file becomes the bottleneck.)
- **Global ledger is month-partitioned.** It inherits the §3.6.1 dated shards
  (`calls-YYYY-MM.jsonl`) and `--since` defaults — never a single unbounded JSONL.
- **No OS job, ever.** cage does not install or manage any launchd/systemd/cron/schtasks
  entry and exposes no `cage scheduler` command. Hands-off automation is the user's own cron
  line calling `cage import` — documented, never installed by cage.
- **Capture is pull-based; surface staleness.** Since nothing runs in the background,
  `cage doctor` and `cage report` show **"last import: N ago"** per the cursor and nudge the
  user to `cage import` / `cage watch` when stale. Docs frame cage as a snapshot-on-demand
  audit tool, not a passive meter.
- **`cage watch`** — optional stdlib **foreground** poll loop (`import_all`; `sleep
  interval`), Ctrl-C clean. Registers nothing; stops with the terminal.
- **Hooks demoted to optional real-time; drop the cwd-`.cage` guard.** Keep the hook command
  strings byte-for-byte unchanged. The guard only existed to stop a global hook from
  scattering stray local `.cage` dirs — obsolete now: `resolve_root` sends a no-project cwd to
  the global `~/.cage` (never creates a stray local dir), so a hook firing anywhere captures
  into global (counts-only, `project`-tagged). This restores **passive capture** for
  hook-firing clients (Claude/Copilot) wherever used; project isolation stays available by
  creating a `.cage/`. Explicit import/export resolve the same way.
- **Honest doctor** — infer each agent's capture state from the **debug heartbeat** (hook
  fired recently ⇒ real-time active; never fired ⇒ warn it likely won't, e.g. under a VS Code
  extension) — do NOT try to detect the client type, which cage can't. Point at
  `cage import`/`cage export` (and optionally `cage watch`) as the universal path, and show
  "last import: N ago."

## 6. Non-negotiables / constraints
- **Style/patterns:** small single-purpose modules (fux spirit); reuse
  `importcmd`/`append_new`/`cage_bin`/`report`/`debuglog`.
- **Use:** stdlib only (`dependencies = []`); stdlib `csv`/`json` for export. **Avoid:** any
  non-stdlib dep, filesystem-watch lib, or network on the capture/read path.
- **No background/OS job:** do not write or register any launchd/systemd/cron/schtasks unit;
  do not add a `cage scheduler` command. The heaviest thing cage runs is a foreground loop
  the user starts and stops.
- **Four agents first-class:** `agents.SURFACES` drives every list; never assume Claude.
- **Additive:** do not remove/alter hooks, MCP, or the project-local `.cage/` ledger.
- **Compliance/safety:** counts-never-content (no prompt bodies in any export);
  `project`/`scope` are basename-only (no full paths); deterministic byte-identical export
  for the same `--since` window; fail-open on capture; idempotent (dedupe by id).
- **Substrate contract:** the only change permitted is **adding one additive optional
  `project` field** (empty = legacy). Update plan §3 + `schema.py`. Do not otherwise touch
  `make_call`/the enums, the attribution/matrix math, or plan §4.4 demo numbers.

None external. Honors `CAGE_LEDGER`, `CAGE_CAPTURE`, `[capture] enabled`. No secrets, no OS
services.

## 8. Edge cases & risks
- No `.cage` anywhere + no `CAGE_LEDGER` → global ledger is created/used; never crash.
- Both a global ledger and a project `.cage/` exist → precedence picks ONE sink per run;
  doctor prints which is active. Never write the same import to two ledgers.
- `cage export` with an empty ledger and `--no-import` → emit an empty-but-valid artifact
  (header row / `[]`), not a crash.
- `export` import step fails (bad log) → fail-open: warn, still export what's in the ledger.
- Re-running import / overlapping with a firing hook or a user's own cron → idempotent no-op.
- Hook fires in a dir with no project `.cage/` → resolves to global `~/.cage` (no stray local
  dir, no scatter); the row is counts-only and `project`-tagged with the cwd basename.
- Malformed `policy.toml` on the capture path → fail open (warn + skip), never a traceback
  (this session hit a raw `tomllib` crash from a duplicate `[debug]`).
- `cage watch` left running → harmless (idempotent); Ctrl-C exits cleanly, leaves nothing
  registered.
- Per-project view for Copilot/Kiro → show total/global, never a per-project number the log
  can't support; doctor/report/export-summary must say so.

## 10. Decisions (formerly open)
- **`cage export` default format = `jsonl`** (lossless, re-ingestable; `report` already
  gives the human summary). `--format json|csv` available.
- **Ledger resolution = `--ledger`/`CAGE_LEDGER` → nearest project `.cage/` → global
  `~/.cage`.** Add a `--ledger` flag to the import/export/report family. Global is **plain
  `~/.cage` on every platform** (greppable, easy to `cd`, mirrors the project-level `.cage/`)
  — **no XDG special-casing**; `CAGE_LEDGER` is the single relocation knob (an XDG user sets
  `CAGE_LEDGER=$XDG_DATA_HOME/cage`).
- **`project` value source = `basename(rec['cwd'])`** read off the transcript (confirmed
  present), not the lossy slug.
- **Incremental import = per-file `(size,mtime)` skip + once-per-run dedupe set** (not
  byte-offset partial parsing).
