# Handoff: output honesty — tokens-default, `—` for unpriced, signal-gated columns, doc generators

**One-liner:** Implement plan Phases 1+2 (+5.6 rider): `report`/`matrix` render tokens by default with `--usd` opt-in, unpriced renders `—` never `$0.0000`, saved/net columns appear only when receipts exist, refusals/footnotes/advice speak once in one voice — all pinned by golden tests that then **regenerate** `docs/cli-output-spec.md`, `docs/formulas.md`, and the bundled policy.toml formula comments (one source, drift-gated).
**Owner / executor:** Claude Code (cage repo)
**Status:** Ready to build
**Stress-tested:** Full debate recorded in `docs/output-and-simplification.plan.md` (Phases 1, 2, 5.6). Survivors that bind: (a) "hide negative numbers" became signal-gating — an unmeasured zero mustn't wear a verdict's clothes, but **a negative net with real receipts renders unconditionally**; (b) `$0.0000`-for-unpriced is the real ambiguity — `—` + the ⚠ block replaces it; (c) matrix token grid never refuses, only its `--usd` cost column explains absence; (d) dollars opt-in on report/matrix only — money-native views (`budget/roi/verdict/compare/estimate`) keep dollars; (e) formulas never become editable config; honesty gates stay constants. Residual risk: this intentionally changes most golden text outputs in one release — mitigated by the spec being the reviewed target *before* code changes, and CSV schemas staying frozen.

## 1. Context & background

Field output from a real project (v0.23 era) showed: `saved $0.0000 / net -$16.11` in a receipt-less window, `$0.0000` cells meaning "unpriceable", a `$0→$0` matrix, duplicate footnotes, an empty `-` agent row. The target renderings for every command × state are already written and reviewed: **`docs/cli-output-spec.md` is the spec of record for this work** — implement to it verbatim.

## 5. Technical approach (decided)

One display-context object (usd on/off · signal flags · gathered footnotes/advice) threaded through the render layer, so gating/dedupe logic lives once, not per view. Generators are separate small build-time tools; the spec generator renders fixture files the golden tests also assert — one artifact. Order inside the run: land render changes + goldens first, generators second (they consume the goldens).
