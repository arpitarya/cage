# Handoff: tool-receipt pricing — a derive-time ladder for call-less token receipts

**One-liner:** Give call-less `unit="tokens"` receipts (graphify/fux tool savings) a USD value via a deterministic resolution ladder — explicit policy routing, else the task/session's dominant model, else loudly UNPRICED — so `roi`/`attrib`/`verdict graphify` show dollars, not just tokens.
**Owner / executor:** Claude Code (cage repo)
**Status:** Ready to build
**Stress-tested:** Debated over two sessions. (a) "Just pick a default model rate" → rejected: a silently guessed rate is the `copilot/auto` sin; the ladder's every rung has a defensible source and the bottom rung is refusal. (b) "Weighted blend across a mixed-model session" → rejected as precision theater on a `modeled` sub-1.0-confidence receipt; dominant model + footnote, deterministic tie-breaks. (c) "Stamp the resolved model onto the receipt at import" → rejected: derive-time only, ledger never rewritten, so a later policy fix re-prices history. Residual risk: a task whose calls are split across models prices at the dominant one — the footnote names it, and `price_at` exists for anyone who cares.

## 1. Context & background

Graphify's interceptor and self-metering emit receipts with `unit="tokens"`, a `task`, but **no `call` id** — the saved tokens belong to future calls the shim can't know. `convert.saved_usd()` prices token receipts at the *linked call's* model (`prices.input_cost_usd`), so an empty call ⇒ empty provider/model ⇒ no price match ⇒ $0. Tokens render; dollars don't. The joins needed to fix this already exist (`taskgroup`'s task-id/session-window resolution).

## 2. Definition of done

- [ ] One resolution helper (new small module or an extension of `convert.py` — executor's call, but ONE place) resolves a pricing `(provider, model, rung)` for a call-less token receipt:
      1. `[tools.<tool>] price_at = "provider/model"` in policy (validated; dangling → treated as no match + warned, mirroring the dangling-alias rule) →
      2. dominant model of the calls sharing the receipt's `task` (fallback: same `session` within the task window) — dominant = max summed `tokens_in` per `(provider, model)`; ties break by total calls, then lexicographic (deterministic) →
      3. no rung matches → unpriced, exactly today's behavior.
- [ ] Receipts **with** a call id take the existing path untouched (byte-identical legacy).
- [ ] `roi`/`attribution`/`verdict` route call-less token receipts through the ladder; rendered USD carries the receipt's own `method` (`modeled`, never upgraded) plus a rung footnote: `priced via [tools.graphify] price_at` or `priced at task model (anthropic/claude-sonnet-4-6)`.
- [ ] Unpriceable tool receipts surface in the UNPRICED ⚠ summary (report/roi) with the fix hint (`set [tools.<tool>] price_at or run in a metered session`).
- [ ] `verdict graphify` names the rung that priced its savings input.
- [ ] `cage query receipt-pricing` explains the ladder with live values; graphify/receipts concept entry extended.
- [ ] CSV outputs carry the rung as a column where the text view footnotes it.
- [ ] Tests exact-number green incl. determinism double-runs; dummyrepo scenario added (S14 or an S5/S7 extension): seeded call-less receipts exercise all three rungs + tie-break; `just test` green.
- [ ] Docs per §9.5. Nothing committed.

## 3. Scope

**In scope:** the ladder, its consumers (roi/attrib/verdict + UNPRICED summary), the `[tools.<tool>] price_at` policy key (read via `policy.load`'s existing deep-merge), and its managed writer: `cage prices route-tool <tool> --to provider/model` — same validation and cage-managed-block write path as `prices alias`, so the UNPRICED fix-hint stays a copy-paste runnable command (the product's established contract; a hand-edit-only pricing key repeats the pre-v0.19 mispricing failure mode). **Fence:** if extending `pricestoml` to the `[tools.*]` table shape proves to be surgery rather than extension, STOP and show the cost — never silently downgrade to a manual-edit hint.
**Out of scope (explicit):** weighted multi-model blending; stamping anything on receipt rows; changing the graphify shim/self-metering emit side; `minutes`/`usd`/`ms`/`gco2` receipt handling (unchanged); **any change to the graphify repo** — savings display stays on cage's surfaces (`verdict graphify` / `attrib` / `roi`), which this work makes dollar-complete.

## 5. Technical approach (decided)

Ladder lives in one function taking `(receipt, calls_by_task_or_session, pol)` → `(usd, rung, model_key) | None`. Consumers pass the same pre-built join structure (build once per view — don't re-scan the ledger per receipt). `convert.saved_usd` keeps its signature for linked receipts; call-less token receipts route through the new helper (executor chooses whether that's inside `saved_usd` or beside it — inside is preferred if the join structure can be threaded without polluting unrelated callers). Derive-time only. Policy key format `provider/model`, validated against `policy.price_match` at use time.

## 6. Non-negotiables / constraints

$0/stdlib · determinism (tie-breaks total-ordered; same ledger+policy ⇒ byte-identical) · **method sacred** (USD inherits the receipt's method, footnoted, never `measured`; rung always visible) · fail-open write path untouched (this is read-side only) · additive-only (no schema change at all in this work) · a wrong number is worse than none (rung 3 refuses) · ledger never rewritten · four agents unaffected · skillgen fragments only for rendered assets · no commits/tags/publishes.

## 7. Dependencies & prerequisites

None. This is a cage-only change; graphify's emit side is untouched and needs nothing.

## 8. Edge cases & risks

- Task with zero calls (graphify ran, agent never called) → session fallback; both empty → rung 3.
- Cross-month task → joins must read via partition-aware `ledger.read_kind` (taskgroup already does).
- Two models exactly tied on tokens_in AND call count → lexicographic winner, tested.
- `price_at` pointing at an unpriced model id → no match, warned in `prices list`/doctor, receipt stays unpriced (never silently $0-at-missing-row).
- Receipt with a call id that doesn't resolve (imported bundle missing that call) → treat as call-less, enter the ladder (this *improves* fleet-merge pricing; test it).
- Historical ledgers: numbers in roi/attrib/verdict will change (tokens now priced) — that's the feature; CHANGELOG says so explicitly.
