# Handoff: stale-wiring liveness — detect + heal orphaned wiring, make dead capture loud

**One-liner:** A wiring artifact whose cage command references a verb the current CLI no longer
accepts fails **silently** (exit 1 to nowhere → unmetered passthrough). Detect every such
artifact against the live parser, heal it on `cage setup`, and make a dead interceptor loud in
`cage doctor` — closing the class of failure behind F1.

**Owner / executor:** Claude Code
**Status:** Ready to build. Design decided; investigation complete.
**Design source:** the stale-wiring investigation (Claude Code, Opus) — its inventory, the
sandbox reproduction, and the parser-liveness prototype are the evidence base. This handoff
locks the two open calls and the scope.

**Stress-tested (proportional — medium change, no contract surface):**
- *Devils-advocate — "parser-liveness will false-positive a valid command as dead."* The parser
  (`cli.build_parser()`) **is** ground truth for "will this exit 1" — it's the same code the CLI
  runs. The round-trip property test (every `verbmap.REMOVED` key maps to a parser-accepted tail)
  pins it. Risk is low and tested.
- *Devils-advocate — "scanning user-level artifacts makes doctor noisy about other projects."*
  A dead verb is rare and unambiguously actionable; severity tiers (`✗` wired vs `·` asset)
  keep signal high. And the whole point is that the real failures here **are** user-level.
- *Pre-mortem — "the heal rewrote a foreign (non-cage) hook."* The foreign-hook-never-touched
  invariant is already load-bearing in every writer; the byte-identity test pins it.
- *Pre-mortem — "the heal embedded a NEW dead verb (a future rename)."* The `REMOVED`→parser
  round-trip property test fails the build if any remediation tail isn't parser-valid.
- **Residual risk:** the `" import"` substring predicate currently heals `import-claude` /
  `import-codex` *by coincidence*. Replacing it with a parser-based predicate must preserve both
  real cases — the single most likely place to silently break capture. Called out in §6/§8 and
  guarded by an explicit test.

## 1. Context & background

v0.28.0 (`048a962`, 2026-07-15) renamed 31 verbs into `verbmap.REMOVED`. Any previously-installed
wiring artifact whose command uses a renamed verb now exits 1 — and because hook/shim stdout
goes nowhere and the shims fail open to exit 0, a dead verb is **indistinguishable from cage
being absent**. Proven for `anton/bin/graphify` (`cage graphify` → unmetered passthrough, 9 days)
and the global `~/.claude/settings.json` SessionStart hook (`cage import-claude`). `doctor`
reports ✅ because `_interceptor` checks existence+PATH, not liveness. This is the root cause
behind F1's empty receipts.

## 5. Decisions (locked — do not re-litigate)

- **Detection = live parser, not grep-`REMOVED`.** `adopt` proves it: dead, installed, and *not*
  in `REMOVED`. Parser = "will it exit 1"; `REMOVED` = replacement hint.
- **Doctor scans user-level artifacts for liveness (no `--machine` flag).** Both real failures
  here are user-level; a liveness check that skips them misses its own reason to exist. Noise is
  controlled by severity tiers, not by hiding the scan. `_portability` stays committed-only.
- **Assets: hash-compare, advisory `·`, `cage doctor` ONLY — never the `report` footer.** Stale
  assets are lower-stakes than a dead hook (agent errors and adapts vs silent zero capture), so
  `·` not `✗`, and keeping it out of the hot path defuses the upgrade-skew nag.
- **Heal extends the existing migrate-and-print path; foreign artifacts never touched; retire the
  `" import"` substring coincidence for a parser predicate, preserving both real cases.**
- **Bundle Fix 3, wiring check above receipts, receipts line qualified when the interceptor is
  dead.**
- **Land before codex removal.**

## 6. Non-negotiables

- **$0 / stdlib-only.** `wiringscan` imports only cage + stdlib; the parser check reuses
  `build_parser()`, no subprocess.
- **Detection is read-only and side-effect-free** — no `cage import`, no execution of any
  artifact (executing can't distinguish "verb dead" from "cage absent" anyway).
- **Foreign (non-cage) artifacts are never modified** — detection may *warn*, heal must not touch.
- **Preserve `import-claude`/`import-codex` healing** when replacing the substring predicate —
  explicit test required.
- **Fail-open** on every capture-path write; new swallow sites logged under `CAGE_DEBUG`
  (`test_debug_coverage` enforces).
- **Doctor is exempt from determinism (already uses a clock), but derived views must stay
  byte-identical** — this change touches none of them; assert it.
- **PII:** counts/commands/paths/hashes only, never content.

## 7. Dependencies

None external. Sequenced **before** codex removal.

## 8. Edge cases & risks

- **The substring-predicate swap** — the #1 silent-break risk. Test `import-claude`/`import-codex`
  still heal, and that a genuine non-import dead verb heals too.
- **A foreign hook containing the literal string "cage"** but not a cage command → never touched;
  test it.
- **`bin/graphify` byte-compare** must not thrash: a correctly-installed shim compares equal and
  is left alone (idempotent).
- **Asset hash on a user who edited their own installed asset** → advisory `·`, not `✗`; accepted.
- **A dead verb in a `.mcp.json`/MCP block** (`mcp` is live) → not flagged; confirm `mcp` stays
  parser-valid.
