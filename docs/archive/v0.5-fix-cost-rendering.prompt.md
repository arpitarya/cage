# Fix prompt ŌĆö Claude Code cost renders as $0 (hand to Claude Code)

> Two compounding bugs make transcript-sourced (Claude Code) spend show as ~$0 in
> `cage report` / `cage budget`, even though the calls + tokens are recorded
> correctly. Fix both. Diagnosed against `anton/.cage` (5,245 `agent="claude-code"`
> calls, 836M input tokens, costed at ~$0).

---

## Bug A (cage code) ŌĆö `report` and `budget` sum a stored `est_cost_usd` that the transcript meter never sets

`cage/transcript.py` records tokens but passes no `est_cost_usd`, so every
Claude Code call row has `est_cost_usd = 0.0`. `cage/report.py` (`g["usd"] +=
c.get("est_cost_usd", 0.0)`) and `cage/budget.py` (same pattern, ~lines 22ŌĆō23) then
sum that zero. `attrib`/`matrix`/`roi`/`human` already recompute from tokens, so
only `report` and `budget` are wrong.

**Fix ŌĆö recompute per-call cost, fall back to stored `est_cost` only when unpriced.**
Add one shared helper (e.g. in `prices.py` or `convert.py`, Ōēż15 lines) and route
`report` + `budget` through it:

```python
def call_usd(pol, call) -> float:
    """Authoritative per-call cost: recompute from tokens x policy when the model
    is priced; else fall back to the stored est_cost_usd (providers cage can't
    tokenize, e.g. a search API that self-reports its cost)."""
    p = policy.price(pol, call.get("provider",""), call.get("model",""))
    if p.get("input") or p.get("output") or p.get("cache_read"):
        return prices.call_cost_usd(pol, call["provider"], call["model"],
            call.get("tokens_in",0), call.get("tokens_out",0), call.get("cached_in",0))
    return float(call.get("est_cost_usd", 0.0))
```

- `report.py`: replace the `est_cost_usd` sum with `call_usd(pol, c)`.
- `budget.py`: same for the day/session sums (it must load policy ŌĆö it may not today).
- Do **not** overwrite the ledger; this is derive-time only (counts stay ground truth).

## Bug B (anton config) ŌĆö provider-name mismatch makes Claude Code unpriceable

The transcript meter stamps `provider="anthropic"`, but `anton/.cage/policy.toml`
keys Claude prices under `claude-sdk` (`[prices.claude-sdk."claude-opus-4-8"]`).
So `policy.price(pol, "anthropic", "claude-opus-4-8")` returns the zero row and the
call prices at $0 even after Bug A is fixed.

**Fix ŌĆö add `anthropic`-keyed price rows to `anton/.cage/policy.toml`**, mirroring
the existing `claude-sdk` ones (the rate is yours to set ŌĆö verify current Opus/
Sonnet/Haiku pricing). Keep the `claude-sdk` rows too (Orff's library calls use
that provider name). Both provider names must exist in the table:

```toml
[prices.anthropic."claude-opus-4-8"]
input = 15.00      # ŌåÉ confirm the real rate before trusting the numbers
output = 75.00
cache_read = 1.50

[prices.anthropic."claude-sonnet-4-6"]
input = 3.00
output = 15.00
cache_read = 0.30

[prices.anthropic."claude-haiku-4-5-20251001"]
input = 1.00
output = 5.00
cache_read = 0.10
```

> Optional hardening (cage code, separate change): make the transcript meter's
> provider configurable (`[transcript].provider` in policy, default `"anthropic"`)
> so a project can align it to its own price-table key instead of editing prices.
