# Task: prevent silent $0 pricing when a model id doesn't exact-match the policy table

## Context

Cage prices each call at derive time via `prices.call_usd` → `policy.price(pol, provider, model)`,
which today does a pure exact-key dict lookup: `pol["prices"][provider][model]`. The
transcript meter (`transcript.py`) stamps `model` **verbatim** from the Claude Code
transcript — i.e. full dated ids like `claude-sonnet-4-5-20250929`,
`claude-opus-4-20250514`. Policies in the wild use short aliases
(`claude-sonnet-4-6`, `claude-opus-4-8`). These never match, so `policy.price`
returns zeros, `call_usd` falls back to `est_cost_usd` (which transcript calls never
set), and the call prices at **$0.00**. Token counts look right; every dollar figure
is wrong. A model you forgot to add is currently indistinguishable from a genuinely
free call — that's the sharp edge to remove.

Read `docs/cage-plan.md` §3.3 (policy) and §5 (metering) and `CLAUDE.md`
(constants/policy/contract three-layer rule, determinism, $0/stdlib) before changing
the pricing path.

## What to build

1. **Family-prefix fallback in `policy.price`.** Exact match first (unchanged). On
   miss, match against keys under the same provider by longest shared model-family
   prefix; the longest matching key wins (deterministic tie-break — never iterate a
   dict in insertion order and take the first). Define "family" deliberately: match
   on a normalized stem so `claude-sonnet-4-5-20250929` resolves to a
   `claude-sonnet-4-5` (or `claude-sonnet-4`) row, but `claude-opus-*` never
   borrows a `claude-sonnet-*` price. Document the normalization rule in a comment
   and, if it's a tunable heuristic rather than contract, put any literal in
   `constants.py` per the three-layer rule — not inline.

2. **Make a fallback observable, never silent.** A family-matched price must be
   distinguishable from an exact match at the read surface. Decide the cleanest
   mechanism (e.g. `policy.price` returns the matched key / a `match="exact|family|none"`
   tag, and `report`/`doctor` surfaces "priced by family" and "UNPRICED" models).
   The non-negotiable: after this change, a model with no exact *or* family match
   must show up as explicitly unpriced in `cage report` / `cage doctor`, not as a
   $0 call hiding in the totals.

3. **Fix the bundled `data/policy.toml` prices** that ship as the default: Opus is
   ~$15/$75 per million (the current `claude-opus-4-8` block at $3/$15 is Sonnet
   pricing). Correct any mislabeled rows and key them to the model strings Claude
   Code actually writes. Do not touch a user's project `.cage/policy.toml`.

## Constraints (cage law — do not violate)

- **$0 / stdlib only.** `dependencies = []`. No new imports outside the stdlib.
- **Determinism.** Same ledger + same policy ⇒ same tables, byte-identical. The
  prefix match must have a total, stable order (longest key, then lexicographic).
- **Three numbers never mixed.** Contract = `schema.py` enums; policy = `policy.toml`;
  heuristics = `constants.py`. A normalization constant goes in `constants.py`.
- **Derive-time only.** Never rewrite the ledger; pricing stays a read-time projection.
- **`method` is sacred.** Don't let a family-matched (approximate) price read as a
  measured invoice if that distinction is exposed anywhere.

## Acceptance

- New tests in `tests/` asserting: (a) exact match still wins; (b) a dated id resolves
  to its family row with the expected per-million numbers; (c) cross-family never
  borrows (opus id does not price off a sonnet row); (d) a totally unknown model
  reports as unpriced rather than $0; (e) longest-prefix tie-break is deterministic.
- `just test` stays green (currently 112 passing); existing plan-number assertions in
  attribution/report/demo tests must not change. `cage demo` still reproduces §4.4.
- `cage report` on a transcript-sourced ledger now shows non-zero, correct cost for a
  current dated Claude model, and flags any unpriced model.

## Out of scope

Don't add network lookups, a model registry download, or fuzzy/edit-distance
matching. Prefix-family + an explicit unpriced signal only.
