# Handoff: `cage prices route-tool` — the managed writer the v0.23 ladder deferred

**One-liner:** Add the managed CLI writer for `[tools.<tool>] price_at` routes so the tool-receipt UNPRICED hint becomes a copy-paste runnable command — closing the one gap between v0.23.0 as shipped and the debated spec.
**Owner / executor:** Claude Code (cage repo)
**Status:** Ready to build
**Stress-tested:** Already debated (recorded in `docs/archive/v0.23-tool-receipt-pricing.handoff.md` §10, resolved 2026-07-14): ship the verb — deciding arguments were the fix-hint contract (every UNPRICED hint in the product is a runnable command, never a hand-edit instruction) and the v0.19 lesson that hand-edited pricing keys are where silent mispricing lives. The v0.23 executor deferred it because it ran from the pre-debate prompt, not because the fence triggered — no new information has weakened the decision. Residual risk unchanged: `pricestoml`'s managed-block machinery may not extend cleanly to the `[tools.*]` table shape — the STOP-and-ask fence carries over.

## 1. Context & background

v0.23.0 shipped the pricing ladder (`cage/receiptprice.py`): rung 1 reads `[tools.<tool>] price_at` from policy, `cage prices list` shows the routes, `cage doctor` flags dangling ones. But the key is hand-edit-only: `receiptprice.UNPRICED_HINT` says `set [tools.<tool>] price_at or run in a metered session`. Everywhere else (`cage prices unpriced`), the hint is a command the user runs. This work adds the writer and upgrades the hint.

## 2. Definition of done

- [ ] `cage prices route-tool <tool> --to <provider>/<model>` writes/updates the project policy's `[tools.<tool>] price_at` via `pricestoml`'s managed-block machinery (mirroring `prices alias`): idempotent re-runs, before/after row printed, bundled policy never touched.
- [ ] Target validated at write time via `policy.price_match` — an unpriced target is written **with a printed warning** (matching how the dangling state is already surfaced by doctor/`prices list`; refusing to write would block the set-route-then-add-price workflow).
- [ ] `cage prices route-tool <tool> --remove` deletes the route (managed-block surgery, idempotent).
- [ ] `<tool>` validated as a single token (the existing label rule); `--to` format `provider/model` validated with a typed `CageError` on garbage.
- [ ] `receiptprice.UNPRICED_HINT` becomes the runnable form: `run: cage prices route-tool <tool> --to <provider>/<model>  (or run in a metered session)` — and the roi/report ⚠ line renders it per-tool with the actual tool name substituted.
- [ ] `cage prices list` output for routes unchanged in substance (it already renders them); `cage query receipt-pricing` and `prices-cli` entries mention the verb; `docs/pricing.md` tool-receipts section swaps its hand-edit instruction for the command.
- [ ] Tests: write/update/remove idempotency, dangling-target warning, typed errors, hint text golden, S14 extended to set its route via the verb instead of writing TOML directly. `just test` green; skillgen regen + `--bless` (the prices fragment changes); determinism double-runs on touched views.
- [ ] CHANGELOG in-tree (notes this completes the v0.23 debated spec); CLAUDE.md pricing bullet edit proposed-not-applied. Nothing committed.

## 3. Scope

**In scope:** the verb (set/update/remove), validation, hint upgrade, docs/query/skillgen touch-ups.
**Out of scope (explicit):** any ladder change (`receiptprice.resolve` untouched); `prices sync` semantics for routes (routes are user intent, never bundled — assert the bundled policy ships none); graphify repo; new rungs or blending.

## 4. Current state

Read first: `cage/pricescmd.py` (the verb family + how `alias` writes), `cage/pricestoml.py` (managed-block text surgery — the extension point and the fence's subject), `cage/receiptprice.py` (`PRICE_AT`, `UNPRICED_HINT`, `routes()`, `dangling()`), `cage/roi.py`/`report` ⚠ rendering, `cage/explain_data.py` (`receipt-pricing`, `prices-cli`), `docs/pricing.md`, `tools/dummyrepo/run.py` S14.

## 5. Technical approach (decided)

Extend `pricestoml` to write a `[tools.<tool>]` table inside the cage-managed block with the same marker discipline as `[prices]`/`[alias]` rows; hand-added routes outside the block are read (policy merge already does) but never rewritten. The verb registers in the existing `prices` subparser group. **Fence (unchanged from the debate):** if the `[tools.*]` shape needs `pricestoml` surgery rather than extension, STOP and show the cost — never silently re-defer.

## 6. Non-negotiables / constraints

Typed `CageError` at the CLI boundary · bundled policy read-only · idempotent writes, byte-identical re-runs · the hint is runnable or the work isn't done · $0/stdlib · no ladder/derive changes beyond the hint string · rendered assets via skillgen only · no commits/tags/publishes.

## 7. Dependencies & prerequisites

None; v0.23.0 is in the tree.

## 8. Edge cases & risks

Route for a tool with zero receipts (fine — takes effect when receipts appear); `--to` pointing at a family-matched (not exact) row — accept, it prices via the normal match path, note in the warning text which match kind resolved; user hand-edited `[tools.<tool>]` outside the managed block then runs the verb — write into the managed block and warn about the shadowed duplicate rather than editing their text (mirror whatever `prices set` does for that case — follow, don't invent).
