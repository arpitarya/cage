# Handoff: pricing freshness — per-commit staleness check + complete Anthropic/OpenAI tables

**One-liner:** Surface a print-only pricing-staleness check on every commit (and on doctor/report for extension users), populate the bundled table with the complete current + recent-history Anthropic and OpenAI lineups (cited), and add a maintainer-side scheduled CI nag so the bundle itself can't silently age.
**Owner / executor:** Claude Code (cage repo)
**Status:** Ready to build
**Stress-tested:** (a) "really verify — fetch vendor pages at runtime" → rejected: no network on any cage code path is law, and a parse-drifted live rate is worse than a stale rate with a cited date; runtime check = local evidence only (version drift, bundle age, UNPRICED presence). (b) "gate the commit when stale" → rejected: cage never blocks (the `verify` precedent); print-only, fail-open. (c) "current models only" → rejected: historical repricing needs recent-history rows too — a June ledger with a since-retired model must still price. (d) "check on every cage command" → rejected as nag-fatigue; per-commit (CLI), plus throttled doctor/report surfacing (extensions). Residual risk: the scheduled CI nag depends on someone acting on the issue it opens — that's the human loop, stated, not papered over.

## 1. Context & background

v0.19 gave policy `[meta] prices_version/prices_date`, `cage prices sync`, and doctor/`prices list` recommendations — but those fire only when the user happens to run them, and only compare project-vs-installed-bundle. Two gaps: nothing checks **at the moment work is committed** (the user's ask: "with every commit, is my pricing current?"), and nothing watches the **bundle's own age** (a user faithfully synced to a 6-month-old bundle is confidently stale). Separately, the bundled table covers a working subset of models; completeness matters because pricing is derive-time — every missing row is a silent $0 in someone's history.

## 2. Definition of done

**A. Per-commit staleness check (print-only, fail-open):**
- [ ] The post-commit path (`gitcommithook`/`hooks.post_commit`) appends a one-line pricing note when — and only when — something is actionable: project meta older than installed bundle → `run cage prices sync`; bundle `prices_date` older than `stale_days` → `bundled prices are N days old — check for a newer cage release`; UNPRICED calls/tool-receipts present in the resolved ledger → the existing runnable hints. Silent when all three are clean. Never a non-zero exit, never blocks, swallow-on-error with a `CAGE_DEBUG` trace (write-path discipline).
- [ ] `stale_days`: policy `[prices] stale_days` with a `constants.PRICES_STALE_DAYS` fallback (default 45; rationale comment) — the established policy-preferred pattern.
- [ ] Extension parity: the same three-part check renders in `cage doctor` (always) and as a report footer note (throttled: only when actionable — no clean-state noise; determinism note below).
- [ ] Determinism guard: the *age* check compares `prices_date` to the newest row `ts` in the ledger (data-relative), **not** the wall clock — derived views stay clock-free. The post-commit hook (a write-path event, clock already allowed there) may use today's date.
- [ ] `cage query prices-freshness` explains all three signals with live values.

**B. Complete Anthropic + OpenAI tables (build-time research, cited):**
- [ ] Web-research the vendors' official pricing pages and populate `data/policy.toml` with every generally-available Anthropic and OpenAI model **plus recent-history models still plausibly present in 2025–26 ledgers** (retired-but-billable-era) — input/output/cache_read per Mtok, one source-URL + retrieval-date comment per row (the file's existing convention).
- [ ] Family aliases/normalizations extended where naming variants exist (the `.`↔`-`/effort-suffix machinery is in place — add rows, don't reinvent matching).
- [ ] `[meta] prices_version/prices_date` bumped; `prices sync` picks the new rows up in a project (tested).
- [ ] A model cage's own fixtures/ledgers reference must exact- or family-match a row (test: run the fixture corpus through `price_match` — zero `none` for anthropic/openai ids).

**C. Maintainer-side freshness nag (repo CI, not a cage code path):**
- [ ] A scheduled GitHub workflow (weekly cron) computes days since the bundled `prices_date` and, past `stale_days`, opens/updates a single pinned issue: "bundled prices are N days old — re-verify against the cited sources." No fetching/parsing of vendor pages (brittle, and a wrong auto-parsed rate is the worst outcome); the workflow nags a human, the human researches, `prices_version` bumps. Idempotent (one issue, updated not duplicated); never touches the publish workflow.

- [ ] Tests: staleness note appears/disappears per signal (goldens); silent-when-clean; hook failure swallowed + debug-logged; stale_days policy override; data-relative age math; fixture-corpus zero-`none`; sync pickup. `just test` green; skillgen `--check`; determinism double-runs (report footer identical across runs on the same ledger).
- [ ] Docs: CHANGELOG in-tree; `docs/pricing.md` freshness section; README one-liner; CLAUDE.md edit proposed-not-applied. Nothing committed.

## 3. Scope

**In scope:** A, B, C above.
**Out of scope (explicit):** any network call in cage itself; commit gating of any kind; auto-applied price updates (`sync` stays user-run; the CI nag stays a nag); non-Anthropic/OpenAI vendors beyond what the bundle already carries (copilot-served rows exist; Kiro/credits unchanged); a price-fetching parser in CI.

## 4. Current state

Read first: `cage/gitcommithook.py` + `cage/hooks.py` (post-commit path — where the note rides), `cage/pricestoml.py`/`policy.py` (`[meta]`, merge), `cage/prices.py` (`price_match`), `cage/doctorcmd.py` (where the sync recommendation lives today), `cage/report.py` (footer/⚠ conventions), `cage/constants.py` (fallback pattern), `data/policy.toml` (row + citation conventions), `.github/workflows/` (publish stays untouched), `tests/fixtures/transcripts/` (the ids that must price).

## 5. Technical approach (decided)

One `freshness(root, pol) -> list[str]` helper (e.g. in `prices.py` or a sibling) returning zero-or-more actionable lines from the three local signals; post-commit, doctor, and report all render the same list (one implementation, three surfaces — the csvout lesson). Post-commit calls it inside the existing fail-open wrapper. CI workflow reads `prices_date` from the bundled TOML with stdlib and uses the GitHub CLI to upsert the pinned issue.

## 6. Non-negotiables / constraints

No network on any cage code path (research is the executor's build-time task; CI nags, never fetches rates) · print-only, exit-0, fail-open on the hook path · derived views stay clock-free (data-relative age) · a wrong number is worse than none (no auto-parsed rates, ever) · bundled policy read-only at runtime · $0/stdlib · publish workflow untouched · skillgen for rendered assets · no commits/tags/publishes.

## 7. Dependencies & prerequisites

None hard. `prices-route-tool` (active pair) touches the same hint strings — run that first or rebase this on it; executor asks if both are pending.

## 8. Edge cases & risks

Empty ledger (new project) → age check has no newest-ts anchor → doctor-only, report silent; project policy with no `[meta]` (pre-v0.19 project) → treated as older-than-bundle with the sync hint; clock skew is irrelevant by design (data-relative); a user who *wants* the note quiet → `stale_days = 0` disables the age signal (documented); CI issue upsert must handle a closed-by-human issue (reopen, don't duplicate).

## 9. Testing & validation

Per §2. Plus one end-to-end in dummyrepo: commit in the testbed with a backdated bundle meta → post-commit output contains the staleness line; sync → line disappears on next commit.
