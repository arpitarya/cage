**Owner / executor:** Claude Code (cage repo)
**Status:** Ready to build
**Stress-tested:** Debate recorded in the plan (Phase 3). Survivors: (a) "deleting commands breaks users" → pre-1.0 is exactly when to break cleanly; an unknown-old-verb error that prints the new name is kinder than a zombie alias, and skillgen regenerates every agent-facing asset in the same change so agents never see old names; (b) "grouping hides power" → nothing is removed from *capability*, only from the front door — every room keeps existing, one group deep; (c) "improve while moving" → forbidden by scope fence: byte-identical outputs modulo the program name in usage lines. Residual risk: muscle memory (yours and the fleet's) — mitigated by the error-with-directions release and the CHANGELOG mapping table.

**Out of scope (explicit):** ANY behavior/output change beyond the program-name line (output-honesty owns rendering); data-compat readers (legacy ledgers keep parsing — the plan's fence); removing `hook-*` callability; MCP tool renames.

## 4. Current state

Read first: the plan Phase 3 block (the spec), `cage/cli.py` + `clicmds.py` (parser + dispatch), `cage/initcmd.py` + `setupcmd.py` (the merge), `cage/importcmd.py` (legacy import-* verbs), every module that emits a `cage …` hint string (grep `"cage "` across `cage/`), `tools/skillgen/fragments/`, `docs/cli-output-spec.md` (headings must match new verbs), `CLAUDE.md`.

## 5. Technical approach (decided)

Parser reorganizes into group subparsers dispatching to the SAME run functions — dispatch signatures untouched, so behavior freeze is structural. The old→new map is one dict driving both the error message and the CHANGELOG table (generated, not hand-duplicated). Hint strings route through one helper (`render.cmd("insights attrib")`) so future moves are one-line — retrofit the emitted strings onto it as part of this work.

## 6. Non-negotiables / constraints

Zero behavior changes (golden old-vs-new diff proves it) · recorded-data readers untouched · error-with-directions exits 1, never runs the moved command silently · every emitted hint runnable post-change (grep-tested) · four agents' rendered assets regenerated in the same change · $0/stdlib · no commits/tags/publishes. This phase ships as its own release.

## 8. Edge cases & risks

Shell scripts/cron lines calling `cage import` — unmoved, safe (deliberate: tier-1 verbs are exactly the ones automation calls) · `cage graphify --` interceptor path moves to `data graphify` — the graphify shim references it: update `runshim`/interceptor text and migration (this is the one behavior-adjacent seam; test the interceptor end-to-end) · argparse prefix-matching may make old verbs "work" accidentally — disable/verify · the one-release error map itself must not shadow a real future verb name.
