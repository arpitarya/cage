# Claude Code prompt: CSV output + agent report-generation skill

You are adding CSV output to cage's read surfaces and teaching the flagship
per-host skill (all four agents) to generate reports/CSVs/summaries on request.
Run AFTER the current tree is reviewed/committed (ask if dirty). **No commits,
tags, pushes, publishes.**

## Context to load first

- `CLAUDE.md`; `cage/render.py` (how views render; the `cage.v1` JSON envelope
  pattern used by `limits --json`); `cage/report.py`, `cage/roi.py`,
  `cage/compare.py`, `cage/study.py`, `cage/calibration.py`, `cage/attribution.py`
  (view shapes); `cage/exportcmd.py`; `tools/skillgen/fragments/` +
  `docs/skillgen.md` (the rendered-assets law); `cage/mcpserver.py`.

## The design (decided)

1. **Two export kinds, never blurred.** The fleet bundle (`cage export
   --study`) stays jsonl — lossless, merge-by-id, re-importable. CSV is a
   REPORTING format: flat, one-way, for spreadsheets. Document the distinction
   in the export help text and the query entry.
2. **`--csv` on read views:** `report`, `attrib`, `roi`, `compare`,
   `study report`, `calibration`, `human`, `trend`. Writes to stdout by
   default (pipe-friendly), `--csv <path>` to a file. Stdlib `csv` module;
   deterministic column order (documented per view); **method/match tags are
   columns, not dropped** — `estimated` must survive into the spreadsheet;
   UNPRICED counts included where the text view shows them. Same numbers as
   the text view by construction (one shared data structure per view feeding
   both renderers — no parallel computation).
3. **Raw row CSV:** `cage export --csv calls|receipts|tasks [--since ...]` —
   flattened ledger rows for pivot-table analysis. Same PII surface as the
   ledger itself (counts, ids, no content). Honors the import-before-export
   toggle if that work has landed.
4. **Skill: reporting recipes (all four hosts).** Extend
   `tools/skillgen/fragments/` with a reporting section so the rendered
   assets (claude SKILL.md · copilot prompt · kiro steering · codex/agents
   skill) teach each agent to handle "generate my cost report / CSV /
   summary":
   - Recipes: weekly spend (`cage report --csv --since 7d`), per-tool savings
     (`cage attrib --csv`), is-tool-X-worth-it (`cage verdict <tool>` + quote
     verbatim), fleet number (`cage study report --csv`), estimate accuracy
     (`cage calibration --csv`).
   - Summarization rules, stated in the skill text: quote cage's numbers
     verbatim; keep method tags and the UNPRICED/observational caveats in any
     summary; never extrapolate or fill gaps — if cage refused
     (INSUFFICIENT DATA / min-n), the summary says so; save CSVs where the
     user asked, default `./cage-report-<view>-<since>.csv`.
   - Render via `python -m tools.skillgen && python -m tools.skillgen
     --bless`; never hand-edit rendered files.
5. **MCP parity:** the read server exposes the same CSV capability (a
   `format: csv` param or equivalent on its report tools) so extension-hosted
   agents without shell access can still produce the CSV content.
6. **`cage query`:** entries for `csv-output` (which views, column contracts,
   text-vs-csv same-numbers guarantee, bundle-vs-csv distinction) and extend
   the skill/agents concept entry with the reporting recipes.

## Task

1. Shared CSV renderer helper in `render.py` (or sibling) — views pass the
   same table structure they render as text; no view computes twice.
2. Wire `--csv` per view (§2) + raw export (§3); typed errors for bad
   combinations (`CageError`).
3. Fragments + regenerate rendered assets for all four hosts (§4).
4. MCP CSV parity (§5). Query entries (§6).
5. Tests: golden CSV per view over the seeded demo ledger (byte-exact,
   determinism double-run); text-vs-csv same-numbers assertion; method-tag
   column present; PII grep on raw CSVs; RFC-4180 quoting (labels/scopes with
   commas can't exist by validation — assert that too); skillgen `--check`
   clean. Dummyrepo: extend a scenario to run one `--csv` view and diff
   byte-exact.
6. Docs: README (reporting section with one CSV example), CHANGELOG in-tree,
   plan-doc note, CLAUDE.md proposed-not-applied.

## Constraints (hard)

- $0/stdlib (`csv` module); determinism (byte-identical CSVs, LF line endings
  regardless of OS — set `newline` explicitly); `method` sacred (tags survive
  into CSV; a spreadsheet must be able to tell measured from estimated).
- One data structure per view feeds text AND csv — if they can disagree, the
  design is wrong.
- Bundle jsonl format untouched; CSV never becomes an import source.
- Four agents: every rendered skill asset gains the recipes — skillgen
  fragments only, never hand-edits.
- No commits; working tree only.
