# Handoff: capture health — make silent zero-capture loud

**One-liner:** When an agent is installed but its log source matched nothing, say so — instead
of letting cage quietly capture zero and print confident totals from the agents that still work.

**Owner / executor:** Claude Code
**Status:** Ready to build.

**Stress-tested:** The failure this targets is real and current: `pathprobe._why()` already
computes the exact diagnosis ("location absent", "no files match \<pattern\>", "parse: 0 rows")
— and then only shows it if the user happens to run `cage doctor --paths`. Nothing volunteers
it. A wrong path produces zero rows, which is **indistinguishable from "I don't use that
agent."** Unlike a stale price (loud: `cage prices unpriced`, ⚠ UNPRICED on
report/overview/compare), nothing tells you.

**What the gate challenged and changed:**

1. **"You're building a false-positive machine."** A nag that fires wrongly trains people to
   ignore the footer — where UNPRICED ⚠, method tags and caveats are load-bearing. One bad
   nag poisons all of them. **This is the design's main risk and it drove the gate.**
   → Survived only via a **triple gate** (§5): agent home exists **AND** zero files matched
   at last import **AND** that agent has never contributed a row. The third clause makes it
   **self-silencing** — once an agent captures one row, it can never nag again. The warning
   can only ever fire for agents that have contributed exactly zero data, which is precisely
   the population where it might be right.
2. **A live filesystem probe in `cage report` — cut.** New I/O on a hot read path, and it
   makes `render_report` impure (it is currently a pure function of its args —
   `clicmds.py:55` reads the state and passes it in). → Replaced by the **`last_import`
   pattern**: `_scan` already computes `exists` and `len(raw)` and already debug-logs them.
   Cache the verdict at import; render from the cache. **Zero new I/O.**
3. **A new state file — cut.** `cursors.json` already carries `_last_import`
   (`importcmd.py:372`), so `_`-prefixed metadata in the cursor map is an established
   precedent. No new file ⇒ no new `cleanup` interaction.
4. **A new opt-out config key — cut.** `[sources.<agent>] replace = true, paths = []` already
   means "disabled by policy" and is already documented in `docs/sources.md` as the way to
   silence a never-installed agent's probe. Reuse it.

**Residual risk to watch:** the user who tried an agent once, then cleaned its logs, gets a
nag until they silence it. Accepted — that user's capture genuinely *is* off for that agent,
and the fix is one documented line. But if this surfaces as noise in practice, the first
lever is tightening the gate, **not** adding a suppression key.

## 1. Context & background

Capture is pull-based: `cage import` sweeps each agent's log location
(`paths.resolve_log_sources` → `importcmd._scan`). When a location is wrong — a vendor moved
its store, a nonstandard install, the `UNVERIFIED-LAYOUT` Windows Kiro path — the sweep finds
nothing and **succeeds**. `cage report` then prints totals from whatever *did* capture, with
no indication that a whole agent is missing.

The diagnosis already exists (`pathprobe._why`), on an opt-in surface nobody runs before they
suspect a problem. This closes the loop: surface it where the user actually looks.

This is independent of `docs/sources-defaults.handoff.md` and arguably higher-value —
that work makes the paths *visible*; this makes a *wrong* path **loud**.

## 2. Definition of done

- [ ] `importcmd` records per-agent capture health into `cursors.json` under a `_health` key
      at every import — derived from facts `_scan` already computes. **No new filesystem I/O.**
- [ ] `importcmd.capture_health(root)` reads it back, exactly paralleling
      `importcmd.last_import(root)` (`importcmd.py:101`).
- [ ] `clicmds.py:55` passes it into `render_report`, exactly paralleling `last_import=`.
      **`render_report` stays a pure function of its arguments.**
- [ ] `report.py` emits one `foot.warn(...)` per triple-gated agent (§5), naming the agent,
      the probed path, and the fix command.
- [ ] `cage doctor` (plain, not `--paths`) surfaces the same verdict.
- [ ] Self-silencing: an agent with ≥1 row in the ledger **never** warns.
- [ ] Self-healing: after the user fixes the path, the next import rewrites `_health` and the
      warning clears with no other action.
- [ ] `[sources.<agent>] replace=true, paths=[]` (disabled by policy) suppresses it — no new
      config key.
- [ ] **CSV never gates:** the warning must not enter any `--csv` output.
- [ ] Report **tables** are byte-identical; only the footer changes.
- [ ] Docs updated (§9.5).

## 3. Scope

**In scope:** `cage/importcmd.py` (`_scan`/`_save_cursors`/a `capture_health` reader),
`cage/clicmds.py:55` (the pass-through), `cage/report.py` (the footer line),
`cage/doctorcmd.py`, `cage/constants.py` (any new literal), `docs/debugging-capture.md`.

**Out of scope — do not build:**
- A new state file. A new config key. A live filesystem probe on any read path.
- Any change to `pathprobe` beyond reuse — `_why()`'s strings are the vocabulary; don't
  fork them.
- Any change to the tables, to CSV, or to a derived number. **Footer only.**
- Auto-fixing a path, guessing a path, or scanning for one. Cage never guesses a location.
- Warning on an agent whose home doesn't exist. Absence of `~/.codex` means "doesn't use
  Codex" and must stay silent forever.

## 5. Technical approach (decided)

**The triple gate.** Warn for agent A only when **all three** hold:

1. **Installed** — A's home marker exists (`~/.claude`, `~/.codex`, `~/.copilot` or the VS
   Code user dir, `~/.kiro` or the Kiro data dir). Absent ⇒ silent forever.
2. **Nothing found** — at the last import, every source for A matched **0 files**.
3. **Never captured** — A has contributed **zero rows** to the ledger, ever.

Clause 3 is what makes this safe: the warning is unreachable for any agent that has ever
worked. Drop it and you have a nag machine.

**The data path** (mirrors `last_import` exactly — do not invent a new one):

```
_scan  → exists, len(raw)  [already computed, already debug-logged]
   ↓
cursors.json["_health"] = {agent: {home: bool, files: int, src: str}}   [_save_cursors]
   ↓
importcmd.capture_health(root)          [parallel to last_import(root)]
   ↓
clicmds.py:55  render_report(..., health=…)   [parallel to last_import=…]
   ↓
report.py  foot.warn(...)                [render_report stays pure]
```

**Message shape** — name the agent, the probed path, and a runnable fix (house pattern: an
UNPRICED warning always carries a runnable fix line):

```
⚠ codex: ~/.codex exists but ~/.codex/sessions matched 0 files — capture is off for this agent.
  cage doctor --paths      (if you don't use codex: [sources.codex] replace=true, paths=[] )
```

**Why not `cage import`'s stdout:** hooks run headless and swallow it. The user looks at
`cage report`. That's where it has to land.

## 6. Non-negotiables / constraints

- **$0 / stdlib only.** `dependencies = []`.
- **Fail-open.** Health recording is best-effort — a failure to write `_health` must never
  break an import. Follow `_save_cursors`'s existing swallow + `debuglog.exception`.
- **`render_report` stays pure.** No filesystem access from `report.py`. State is read in
  `clicmds` and passed in. This is the existing contract; do not break it.
- **Determinism.** Same ledger + same policy ⇒ same **tables**. The footer may vary with
  machine state (`_last_import` already does) — the tables may not. Test byte-identity.
- **CSV never gates.** No warning text in any CSV row. `—` never enters CSV data.
- **Four agents, always** — claude · codex · copilot · kiro each get a home marker.
- **Never guess a path.** Report the miss; never scan for or infer a location.
- **Do not touch:** `pathprobe._why()`'s strings (reuse); the `NEVER` list; the tables.
- **PII:** paths only, never content. `_health` stores a path + counts — no filenames.

## 8. Edge cases & risks

- **Fresh install, never imported** → no `_health` yet ⇒ no warning. Correct: nothing is
  known. `cage doctor` should fall back to a **cheap live probe** (`home.exists()` + a
  bounded `next(iter(src.glob(pattern)), None)` — **do not** call `pathprobe._parse_rows`,
  which parses every transcript).
- **Copilot has two sources** (CLI + VS Code) with two different homes. Gate 2 requires *all*
  its sources matched 0. Gate 1 should pass if *either* home exists. Get this right or
  Copilot nags every CLI-only user.
- **Kiro is a file source**, not a dir — `_scan` takes `raw = [src]` when `src.is_file()`, so
  `len(raw)` is 1 even for an empty file. Gate 2 must mean "no data", not "no file". Kiro's
  log is coarse by design (`generatedTokens` often 0) — do **not** let that read as broken.
- **`disabled by policy`** (`replace=true, paths=[]`) ⇒ the agent has no sources at all ⇒ must
  be silent, not "0 files matched". Check `SourcesResolution.disabled`.
- **Health goes stale** if imports stop. Bounded by the existing `_last_import` staleness
  advice, which already fires. Don't build a second staleness mechanism.
- **`cleanup`'s orphan-cursor prune** must not eat `_health`. `_last_import` survives today —
  confirm why, and inherit it. Add a test.

## 9. Testing & validation

- **New `tests/test_capture_health.py`:**
  - each gate independently suppresses the warning (home absent · files found · agent has rows)
  - all three true ⇒ exactly one warning, naming agent + path + fix
  - **self-silencing:** seed one row for the agent ⇒ silent
  - **self-healing:** 0 files → warn; re-import with files present → cleared
  - `disabled by policy` ⇒ silent
  - copilot with CLI home only and CLI files present ⇒ silent (the §8 trap)
  - kiro file-source: present-but-empty ⇒ correct verdict
  - `_health` survives `cleanup`
  - health write failure ⇒ import still succeeds (fail-open)
- **Purity test:** `render_report` called with a fixed `health=` dict produces identical
  output regardless of the filesystem.
- **Table byte-identity:** report/attrib/matrix tables unchanged with and without a warning.
- **CSV:** no warning text in any `--csv` output for any warning state.
- `tests/test_debug_coverage.py` must stay green — a new swallow site needs a `CAGE_DEBUG`
  log ("fail-open but never silent" is tested, not aspirational).
- Golden fixtures: a new report footer state ⇒ **re-bless**
  (`CAGE_BLESS_GOLDENS=1 pytest tests/test_output_spec.py`) then
  `python -m tools.docgen --target spec`. A shipped output change without a regenerated spec
