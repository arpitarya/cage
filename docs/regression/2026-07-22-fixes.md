# Cage fixes — from the 2026-07-22 capture report

Actionable checklist derived from [`2026-07-22-capture-report.md`](2026-07-22-capture-report.md),
which analyzed the real `~/.cage` ledger (36,451 calls). Ordered by impact on cage's
value proposition. Each item names the likely code site so it can be picked up
directly. Nothing here is a crash — cage is fail-open; these are capture/attribution
gaps and misleading signals.

## P0 — attribution has no real data

- [ ] **Receipts never captured in practice.** 3 receipts total, all the `cage demo`
      seed (`task=fix-handover-bug`); 0 real across 36,451 calls. Attribution/matrix/
      roi/verdict therefore run on demo data.
  - Verify `cage setup` puts the graphify interceptor (`bin/graphify` / `.cage/bin/cage-run`)
    ahead of the real binary on PATH, and that `cage data graphify -- …` is the invoked path.
  - Confirm the fux `cage_receipt.py`, `responsecache.py`, and `compress.py` emit + push
    receipts to the resolved ledger (not a stale/none sink).
  - Add a **`cage doctor` check**: "receipts: 0 captured — attribution has no data",
    made loud the same way UNPRICED is (`doctorcmd.py`).
  - Sites: `graphifymeter.py`, `metering.record_receipt`, `responsecache.py`, `compress.py`, `doctorcmd.py`.

## P1 — capture-health misreports

- [ ] **`_health.captured` conflates "nothing new this run" with "broken".** codex/copilot/
      kiro read `captured:false` while holding 373/60/16 rows.
  - Split into `captured_this_run` **and** `last_captured_ts` / `rows_total`.
  - Base the doctor "installed but capturing nothing" ⚠ on `files_present AND rows_total==0`,
    never on a single run's delta.
  - Sites: `importcmd.py` (writes `_health`), `freshness.py` / capture-health, `doctorcmd.py`.

- [ ] **Kiro capture is empty** (16 calls, 198 input tokens, 0 output) from the macOS
      Library log. Distinguish "kiro log found but ~0 tokens" from healthy agents; recommend
      the proxy fallback (`cage meter -- <cmd>`). Verify the Library path is current for both
      VS Code Kiro and CLI Kiro. Sites: `paths.py` (kiro_data), `doctorcmd.py`, `kirowire.py`.

## P2 — pricing & reporting clarity

- [ ] **`copilot/auto` UNPRICED** (24 of 60 copilot calls, 975k tokens → $0). Ship a default
      alias for the router pseudo-model in bundled `data/policy.toml` with an "approximate —
      auto is a router" note, or make `cage prices` prompt for it. Sites: `data/policy.toml`, `pricescmd.py`.

- [ ] **Headline token/$ is 98% cache reads.** Add a cache-efficiency line/column to
      `report --usd` (cached vs fresh tokens, cache-read share of cost) so the number is honest
      at a glance. Sites: `report.py`, `humanview.py`.

- [ ] **`gap_ms` on ~1% of rows** — verify `transcript.parse` stamps it across the whole
      claude transcript; log when skipped (missing previous-turn ts). Sites: `transcript.py`.

## P3 — observability (the "more debugging" ask)

- [ ] **No `debug.log` ever written** (CAGE_DEBUG never on) → no trail for the above.
  - Add a **small always-on** `state/capture.log` breadcrumb: one line per import run per
    agent — `agent · files_seen · rows_parsed · rows_new_after_dedupe · rows_total · resolved_src`
    (counts-only, PII-safe, cleanup-allowlisted).
  - Keep the verbose trail (per-file skip reasons, receipt emit/skip, parser mismatches)
    behind `CAGE_DEBUG=1`.
  - Include `capture.log` in `cage doctor --bundle`.
  - Sites: `debuglog.py`, `importcmd.py`, `transcript.py`, `doctorbundle.py`, `cleanup.py` (allowlist).
